    1. In decomposer.py, add EXPANSION_PROMPT_TEMPLATE that takes a single task's details plus its expansion_prompt (from ComplexityAnalysis) and asks Claude to produce a list of sub-TaskSpec objects. The prompt should instruct:
       - Sub-tasks must target specific files (one per sub-task)
       - Sub-tasks should form a sequential dependency chain
       - Each sub-task description should be specific and actionable
       - The expansion_prompt from complexity analysis is the primary guidance

    2. Add a wrapper model for expansion output (or reuse TaskPlan — but a simpler model may be better):
       ```python
       class ExpansionResult(BaseModel):
           subtasks: list[TaskSpec]
       ```
       Add to models.py if needed, or define locally in decomposer.py.

    3. Add `async def _expand_task(self, task: TaskSpec, analysis: ComplexityAnalysis) -> list[TaskSpec] | None`:
       - Builds prompt from EXPANSION_PROMPT_TEMPLATE with task details + analysis.expansion_prompt
       - Makes SDK query() call with ExpansionResult schema
       - On success: assigns sequential IDs ("{task.id}.1", "{task.id}.2", ...), sets requires chain (each depends on previous), inherits parent role
       - On failure: returns None (task will pass through unchanged)

    4. Add `async def _expand_complex_tasks(self, plan: TaskPlan, analyses: list[ComplexityAnalysis]) -> TaskPlan`:
       - Build a map of task_id -> ComplexityAnalysis
       - For each task in plan.tasks:
         - If complexity_score > self._complexity_threshold AND analysis exists for this task:
           - Call _expand_task(task, analysis)
           - If expansion succeeds: replace task with sub-tasks in the task list
           - If expansion fails: keep original task
         - Else: keep original task unchanged
       - **Dependency rewiring**: After all expansions, scan all tasks. For any task T whose `requires` contains an expanded task_id X, replace X with the last sub-task ID of X (e.g., "X.3"). Build a mapping of original_id -> last_subtask_id for this.
       - Return new TaskPlan with expanded tasks list, same feature_name and max_agents

    5. Wire into decompose():
       - After _analyze_complexity() succeeds and scores are populated:
       - Call _expand_complex_tasks(plan, analyses)
       - Return the expanded plan
       - If _analyze_complexity returned None, skip expansion entirely

    6. Ensure the full pipeline is: decompose (SDK call 1) -> analyze_complexity (SDK call 2) -> expand each complex task (SDK call 3 per task). Three distinct phases, each a separate SDK call.
