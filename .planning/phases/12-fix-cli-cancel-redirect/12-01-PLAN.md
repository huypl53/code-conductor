<objective>
Fix the `cancel_agent` signature mismatch between CLI dispatch and Orchestrator implementation so that CLI cancel and redirect commands execute without TypeError.

Purpose: The CLI already calls `cancel_agent(agent_id)` and `cancel_agent(agent_id, new_instructions="...")` but the Orchestrator expects `cancel_agent(agent_id, corrected_spec: TaskSpec)`. This causes runtime TypeErrors for both cancel and redirect commands. The fix changes the Orchestrator to accept the CLI's call shape and internally reconstruct the TaskSpec from state.

Output: Fixed `cancel_agent` method + integration tests validating both cancel and redirect paths through the actual implementation (not just mocked dispatch).
</objective>

    ```python
    async def cancel_agent(self, agent_id: str, corrected_spec: TaskSpec) -> None:
    ```
    With:
    ```python
    async def cancel_agent(self, agent_id: str, new_instructions: str | None = None) -> None:
    ```

    Replace the method body after the asyncio.Task cancellation block (lines 397-401) with state-aware spec reconstruction:

    1. Read state via `state = await asyncio.to_thread(self._state.read_state)` (same pattern used in `resume()` and throughout orchestrator).
    2. Find agent record: `agent_rec = next((a for a in state.agents if a.id == agent_id), None)`.
    3. Get task_id from agent_rec, then find task: `task = next((t for t in state.tasks if t.id == task_id), None)`.
    4. If `task is None`, return early (no spec to reconstruct, nothing to re-spawn).
    5. Build `corrected_spec = TaskSpec(id=task.id, title=task.title, description=new_instructions if new_instructions else task.description, role=agent_rec.role if agent_rec else "developer", target_file=task.target_file or "", material_files=task.material_files, requires=task.requires, produces=task.produces)`.
    6. Create and store the new asyncio.Task as before: `asyncio.create_task(self._run_agent_loop(corrected_spec, sem))`.

    Update the docstring to reflect the new parameter. Keep the COMM-05 reference.

    IMPORTANT: Use `task.target_file or ""` to guard against None (Task.target_file is `str | None` but TaskSpec.target_file is `str`).

    No changes to `input_loop.py` — the CLI already calls the correct API shape.
