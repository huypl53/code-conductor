    - test_stream_done_finalizes: Create a streaming AssistantCell, start streaming, append a token, then call finalize(). After finalize, _is_streaming should be False and _stream should be None.
    - test_status_footer_token_update: Mount a StatusFooter. Post TokensUpdated({"input_tokens": 100, "output_tokens": 50}) to the app. The footer's token_count reactive should become 150 and the left label should contain "tokens: 150".

  <action>
    **1. Add StreamingStarted message to messages.py:**
    Add a `StreamingStarted` message class (no fields needed — it signals that an AssistantCell should be created in streaming mode).

    **2. Rewrite AssistantCell in transcript.py to support streaming lifecycle:**
    - AssistantCell gets two modes: static (existing — takes text arg, renders as Static) and streaming (no text arg, starts with LoadingIndicator).
    - Add `__init__(self, text: str | None = None)` — if text is None, it's in streaming mode.
    - In `compose()`: if streaming mode, yield `Static("Conductor", classes="cell-label")` then `LoadingIndicator()`. If static mode, yield the existing Static content.
    - Add `_is_streaming: bool` attribute (True when in streaming mode and not yet finalized).
    - Add `_stream: MarkdownStream | None = None` attribute.
    - Add `async def start_streaming(self) -> None`: removes the LoadingIndicator, mounts a `Markdown("")` widget, calls `Markdown.get_stream(self._markdown)` to get a MarkdownStream, stores it in `self._stream`.
    - Add `async def append_token(self, chunk: str) -> None`: calls `await self._stream.write(chunk)` if stream exists.
    - Add `async def finalize(self) -> None`: calls `await self._stream.stop()`, sets `self._stream = None`, sets `self._is_streaming = False`.
    - CRITICAL: Use `Markdown.get_stream()` with `write()`, NOT `append()`.

    **3. Add add_assistant_streaming() to TranscriptPane:**
    - Add `async def add_assistant_streaming(self) -> AssistantCell`: creates a streaming AssistantCell (no text), mounts it, scrolls to bottom, returns the cell reference.

    **4. Rewrite StatusFooter with reactive attributes:**
    - Import `reactive` from `textual.reactive`.
    - Add reactive class attributes: `model_name: reactive[str] = reactive("--")`, `mode: reactive[str] = reactive("interactive")`, `token_count: reactive[int] = reactive(0)`, `session_id: reactive[str] = reactive("--")`.
    - Add `watch_model_name`, `watch_mode`, `watch_token_count`, `watch_session_id` methods that all call `_refresh_display()`.
    - `_refresh_display()` updates the `#status-left` Static with formatted string: `"model: {self.model_name} | mode: {self.mode} | tokens: {self.token_count} | session: {self.session_id}"`.
    - Add `on_tokens_updated(self, event: TokensUpdated) -> None` handler: extracts `input_tokens + output_tokens` from `event.usage` dict, sets `self.token_count`.
    - Update `compose()` to use the reactive values in the initial render (call `_refresh_display` in `on_mount`).

    **5. Write tests in test_tui_streaming.py:**
    Write the 5 tests listed in `<behavior>` using Textual's `run_test()` headless pattern. Each test creates a minimal app that mounts the widget under test, posts messages, and asserts widget state. Follow the inline `run_test()` pattern established in test_tui_shell.py (no fixtures).

    For AssistantCell tests, create a small test app that mounts a single AssistantCell. For StatusFooter tests, create a test app that mounts a StatusFooter and posts messages.
