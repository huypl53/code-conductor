    - test_dispatch_unknown: _dispatch_command("blah", orchestrator) does NOT call any orchestrator method, prints "Unknown command" via console

  </behavior>
  <action>
    1. Create `packages/conductor-core/src/conductor/cli/input_loop.py`:

       a. `async def _ainput(prompt: str = "") -> str`: Returns `await asyncio.to_thread(input, prompt)`.

       b. `async def _dispatch_command(line: str, orchestrator, state_manager=None, console=None) -> bool`:
          Parse the line by splitting on whitespace. First token is the command.
          - `cancel <agent_id>`: Call `await orchestrator.cancel_agent(agent_id)`. Print confirmation.
          - `feedback <agent_id> <message...>`: Call `await orchestrator.inject_guidance(agent_id, " ".join(rest))`. Print confirmation.
          - `redirect <agent_id> <new_instructions...>`: Call `await orchestrator.cancel_agent(agent_id, new_instructions=" ".join(rest))`. Print confirmation.
          - `status`: If state_manager provided, read state via `asyncio.to_thread(state_manager.read_state)` and print `_build_table(state)` to console. If no state_manager, print "No state manager available".
          - `quit` / `exit`: Return `True` (signals loop exit).
          - Anything else: Print "Unknown command: {cmd}. Available: cancel, feedback, redirect, status, quit" to console.
          - Return `False` for all commands except quit/exit.
          - Use `rich.console.Console(stderr=True)` as default console if none provided — prints outside the Live context without corrupting it.

       c. `async def _input_loop(human_out: asyncio.Queue, human_in: asyncio.Queue, orchestrator, state_manager=None, console=None) -> None`:
          Use `asyncio.wait` pattern from research (Pattern 5 / Pitfall 3):
          - Create two tasks: `input_task = asyncio.create_task(_ainput("> "))` and `queue_task = asyncio.create_task(human_out.get())`.
          - `await asyncio.wait({input_task, queue_task}, return_when=asyncio.FIRST_COMPLETED)`.
          - If `queue_task` completed: Extract `HumanQuery`, print `[bold yellow]Agent question:[/] {query.question}` via console, read answer with `_ainput("Your answer: ")`, put answer into `human_in`. Recreate queue_task.
          - If `input_task` completed: Strip line, if non-empty call `_dispatch_command`. If returns True (quit), cancel pending tasks and return. Recreate input_task.
          - Cancel all pending tasks on exit (CancelledError from gather will propagate here too — handle gracefully with try/except CancelledError).

    2. Add tests to `tests/test_cli.py`:
       - All dispatch tests use a `MagicMock` orchestrator with `cancel_agent` and `inject_guidance` as `AsyncMock`.
       - `test_dispatch_cancel`: Calls `_dispatch_command("cancel agent-1", mock_orch)`, asserts `mock_orch.cancel_agent.assert_awaited_once_with("agent-1")`.
       - `test_dispatch_feedback`: Calls `_dispatch_command("feedback agent-1 great work on the models", mock_orch)`, asserts `mock_orch.inject_guidance.assert_awaited_once_with("agent-1", "great work on the models")`.
       - `test_dispatch_redirect`: Calls `_dispatch_command("redirect agent-1 work on auth instead", mock_orch)`, asserts `mock_orch.cancel_agent.assert_awaited_once_with("agent-1", new_instructions="work on auth instead")`.
       - `test_dispatch_unknown`: Calls `_dispatch_command("blah", mock_orch)`, asserts no orchestrator methods called.
       - `test_dispatch_status`: Calls `_dispatch_command("status", mock_orch, state_manager=mock_sm)` where mock_sm.read_state returns a ConductorState with one task. Asserts read_state was called.
