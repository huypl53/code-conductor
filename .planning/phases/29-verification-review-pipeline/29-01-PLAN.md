    - VerificationResult has three bool fields: exists, substantive, wired
    - verify() on missing file returns exists=False, substantive=False, wired=False
    - verify() on file with only `pass` in function body returns exists=True, substantive=False
    - verify() on file with `raise NotImplementedError` returns exists=True, substantive=False
    - verify() on file with `# TODO: implement` returns exists=True, substantive=False
    - verify() on file with `return None` as only function body returns exists=True, substantive=False
    - verify() on file with real implementation returns exists=True, substantive=True
    - verify() on file imported by another file returns wired=True
    - verify() on file not imported anywhere returns wired=False
    - stub_patterns is a configurable list of regex strings passed to constructor
    - DEFAULT_STUB_PATTERNS is exported as a module-level constant
  </behavior>
  <action>
Create `packages/conductor-core/src/conductor/orchestrator/verifier.py`:

1. Define `VerificationResult(BaseModel)` with fields: `exists: bool`, `substantive: bool`, `wired: bool`, plus `stub_matches: list[str]` (which stub patterns matched, for debugging).

2. Define `DEFAULT_STUB_PATTERNS` as a list of regex strings:
   - `^\s*pass\s*$` (pass-only line)
   - `raise\s+NotImplementedError` (NotImplementedError)
   - `#\s*TODO` (TODO markers, case-insensitive flag)
   - `^\s*return\s+None\s*$` (empty return None)
   - `^\s*return\s*$` (bare return)
   - `\.\.\.\s*$` (ellipsis body)

3. Define `TaskVerifier` class:
   - `__init__(self, repo_path: str, stub_patterns: list[str] | None = None)` — stores repo_path and compiles stub_patterns (defaults to DEFAULT_STUB_PATTERNS).
   - `async def verify(self, target_file: str) -> VerificationResult`:
     - Check file exists (Path(repo_path) / target_file). If not, return all False.
     - Read file content. Check each stub pattern against content using `re.search` (with MULTILINE). If ANY pattern matches AND the file has fewer than 10 non-empty non-comment lines of actual code, mark substantive=False. The logic: stub patterns alone are insufficient (a real file might have a TODO comment in addition to real code), so combine pattern match with a line-count heuristic. If no stub patterns match OR the file has >= 10 substantive lines, mark substantive=True.
     - For wiring: use `asyncio.to_thread` to run `subprocess.run(["grep", "-rl", "--include=*.py", "--include=*.ts", "--include=*.js", target_basename, repo_path], capture_output=True, text=True)`. Filter out the target file itself from results. If at least one other file references the basename, wired=True.

Write tests first (RED), then implement (GREEN). Tests should use `tmp_path` fixture to create real files on disk. For wiring tests, create a second file that imports the target.
