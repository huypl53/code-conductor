<objective>
Wire the SDK stream loop in `_stream_response` to detect `conductor_delegate` tool-use events
and produce two visible outcomes: (1) the active AssistantCell label changes to "Orchestrator — delegating",
and (2) an OrchestratorStatusCell is mounted with the task description from accumulated input_json_delta.

Without it, users see a blank "Assistant" cell during delegation with no indication of what the
orchestrator is doing.

    - ORCH-01: After label mutation, the AssistantCell visually reads "Orchestrator — delegating" (not "Assistant")
    - ORCH-02: On content_block_stop for conductor_delegate, DelegationStarted message is posted; TranscriptPane.on_delegation_started mounts an OrchestratorStatusCell with the parsed task description

  <implementation>
    RED phase: Create test_tui_stream_interception.py with tests for all 4 behaviors. Tests should:
    - Use inline TestApp pattern (not fixtures) per Phase 44 convention
    - Mock StreamEvent with dict-based event payloads simulating the content_block_start -> content_block_delta -> content_block_stop sequence
    - Test label mutation on AssistantCell via query_one(".cell-label", Static).content
    - Test OrchestratorStatusCell creation and task description content
    - Test empty/malformed JSON fallback for input_json_delta accumulation
    - Test that multiple content_block_index values don't collide (indexed buffer)

    GREEN phase — two changes:

    1. In app.py `_stream_response`, add tool-use state machine AFTER the existing text_delta handling:
       - Initialize local `_tool_input_buffers: dict[int, list[str]]` and `_tool_use_names: dict[int, str]` at top of method (before loop)
       - On content_block_start with content_block.type=tool_use: record name and init buffer by index; if name is conductor_delegate and cell is not None, mutate label via cell.query_one(".cell-label", Static).update("Orchestrator — delegating") wrapped in try/except
       - On content_block_delta with delta.type=input_json_delta: append partial_json to buffer by index
       - On content_block_stop: pop name and buffer by index; if name is conductor_delegate, json.loads the joined buffer (with try/except JSONDecodeError fallback to {}), extract task description from args.get("task", "delegating..."), post DelegationStarted message
       - Import json at top; import DelegationStarted from conductor.tui.messages inside the method (lazy import pattern)

    2. In transcript.py TranscriptPane, add on_delegation_started handler:
       - async def on_delegation_started(self, event: DelegationStarted) -> None
       - Create OrchestratorStatusCell(label="Orchestrator — delegating", description=event.task_description)
       - Store as self._orch_status_cell (single ref, type: OrchestratorStatusCell | None, initialized to None in __init__)
       - await self.mount(cell) then self._maybe_scroll_end()
       - Import DelegationStarted lazily inside handler body

    CRITICAL anti-patterns to avoid:
    - Do NOT read content_block_start.content_block.input for tool input — always {} in streaming
    - Do NOT use await mount() inside _stream_response — use post_message
    - Do NOT use a single buffer variable — must be dict keyed by content_block_index
    - Do NOT mutate label on content_block_stop — do it on content_block_start for immediate feedback
