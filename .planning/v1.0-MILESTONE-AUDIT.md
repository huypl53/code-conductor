## Integration Findings

### HIGH: Dashboard cancel/redirect type mismatch

**File:** `packages/conductor-core/src/conductor/dashboard/server.py` lines 86-112
**Bug:** Passes `TaskSpec(...)` as second arg to `cancel_agent()`; signature requires `str | None`.
**Fix:** Replace `TaskSpec(...)` with `None` (cancel) or `message` string (redirect).
**Affects:** COMM-05, DASH-06

### HIGH: Production WebSocket URL mismatch

**File:** `packages/conductor-dashboard/src/App.tsx` line 11
**Bug:** `WS_URL` uses `window.location.host` — resolves to sirv port (4173), not FastAPI port (8000).
**Fix:** Add runtime backend URL configuration (env var, query param, or config endpoint).
**Affects:** PKG-02, all DASH requirements

### MEDIUM: Pause never sets WAITING status

**File:** `packages/conductor-core/src/conductor/orchestrator/orchestrator.py`
**Bug:** `pause_for_human_decision` never sets `AgentRecord.status = WAITING` in state.
**Fix:** Add state mutation before human await.
**Affects:** COMM-07, DASH-04

### MEDIUM: AgentRecord.status stuck at WORKING

**File:** `packages/conductor-core/src/conductor/orchestrator/orchestrator.py`
**Bug:** `_make_complete_task_fn` updates Task.status but not AgentRecord.status.
**Fix:** Add `AgentRecord.status = DONE` mutation on task completion.
**Affects:** CORD-01, DASH-01

### LOW: Test masks wrong contract

**File:** `tests/dashboard/test_server_interventions.py` lines 115-120
Asserts TaskSpec-passing as correct behavior, masking the type mismatch.
