<feature>
  <name>StateManager: File-Locked Atomic State Read/Write</name>
  <files>packages/conductor-core/src/conductor/state/manager.py, packages/conductor-core/tests/test_state.py</files>
  <behavior>
    RED phase — write these tests first, all must fail:

    CORD-01 tests:
    - test_task_round_trip: Create StateManager with tmp_path, mutate to add a Task, read_state(), assert task fields match (id, title, description, status="pending")
    - test_full_state_round_trip: Add a Task, AgentRecord, and Dependency via mutate, read_state(), assert all three lists have 1 item each with correct field values
    - test_empty_state_on_missing_file: read_state() on non-existent file returns ConductorState with empty lists and version="1"

    CORD-02 tests:
    - test_assign_task: Add a task via mutate, then call assign_task(task_id, agent_id), read_state(), assert task.assigned_agent == agent_id and task.status == "in_progress"
    - test_concurrent_writes_no_corruption: Spawn two multiprocessing.Process workers, each writing 10 tasks (agent_id prefix differs), join both, read final state, assert len(tasks) == 20 (no writes lost). Use start_method="spawn" context.

    CORD-03 tests:
    - test_update_task_status: Add a task, call update_task_status(task_id, TaskStatus.COMPLETED, output={"result": "done"}), read_state(), assert task.status == "completed" and task.outputs == {"result": "done"}
    - test_orchestrator_observes_status: Add a task, assign to agent, update status to COMPLETED, create a second StateManager instance pointing to same file, read_state() from second instance, assert it sees the completed status

    CORD-06 tests:
    - test_all_tasks_visible: Add 3 tasks with different agents, read_state(), assert all 3 tasks visible with correct assignments

    GREEN phase — implement StateManager:
    - __init__(self, state_path: Path): store path, derive lock_path as state_path.with_suffix(".json.lock")
    - read_state(self) -> ConductorState: if file doesn't exist return ConductorState(), otherwise model_validate_json(read_text()). Wrap parse errors in StateCorrupted.
    - mutate(self, fn: Callable[[ConductorState], None], timeout: float = 10.0) -> ConductorState: acquire FileLock with timeout, read state, apply fn, atomic write (mkstemp in same dir, write, fsync, os.replace), release lock. Catch filelock.Timeout -> raise StateLockTimeout.
    - _atomic_write(self, data: str) -> None: mkstemp(dir=state_path.parent), write data, fsync, os.replace to state_path. Clean up temp on failure.
    - assign_task(self, task_id: str, agent_id: str) -> None: mutate to set task.assigned_agent and task.status to IN_PROGRESS
    - update_task_status(self, task_id: str, status: TaskStatus, output: dict | None = None) -> None: mutate to update status and optionally merge output

    Also update __init__.py to re-export StateManager.

    REFACTOR phase: clean up if needed, ensure all tests still pass.
  </behavior>
  <implementation>
    Follow research Pattern 1 (File-Locked Atomic Read-Modify-Write) exactly. Key constraints:
    - Lock file MUST be in same directory as state.json (same filesystem for os.replace)
    - tempfile.mkstemp MUST use dir=state_path.parent (not system /tmp)
    - Always fsync before os.replace for durability
    - Always clean up temp file on failure (os.unlink in except block)
    - FileLock timeout MUST be set (default 10.0s), never -1
    - Catch filelock.Timeout and re-raise as StateLockTimeout
    - Catch pydantic ValidationError in read_state and re-raise as StateCorrupted
    - For concurrent test: use multiprocessing.get_context("spawn") to avoid fork() fd inheritance issues
    - The worker function for concurrent test must be a module-level function (not nested) for pickle compatibility with spawn
  </implementation>
</feature>
