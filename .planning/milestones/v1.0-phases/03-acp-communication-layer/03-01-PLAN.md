1. Create `src/conductor/acp/permission.py` with:

```python
class PermissionHandler:
    """Routes sub-agent tool permission requests and AskUserQuestion calls.

    Wraps all async decision logic in asyncio.wait_for to enforce timeout.
    On timeout, returns PermissionResultDeny (safe default = deny).
    """

    def __init__(
        self,
        *,
        timeout: float = 30.0,
        answer_fn: Callable[..., Awaitable[PermissionResultAllow | PermissionResultDeny]] | None = None,
        state_manager: StateManager | None = None,
    ) -> None:
        ...
```

Key implementation details:
- `handle(tool_name, input_data, context)` is the `can_use_tool` callback signature matching SDK expectations: `(str, dict, ToolPermissionContext) -> PermissionResultAllow | PermissionResultDeny`
- Wrap `_route(tool_name, input_data)` in `asyncio.wait_for(..., timeout=self._timeout)`
- On `asyncio.TimeoutError`, return `PermissionResultDeny(message=f"Permission timeout after {self._timeout}s — denied by safe default")`
- `_route()`: if `tool_name == "AskUserQuestion"` -> call `self._answer_fn(input_data)`, else -> return `PermissionResultAllow(updated_input=input_data)`
- Default `_answer_fn` when `state_manager` is provided: read state via `asyncio.to_thread(self._state_manager.read_state)`, build answers dict with "proceed" for each question, return `PermissionResultAllow(updated_input={"questions": ..., "answers": ...})`
- Default `_answer_fn` when no state_manager: return "proceed" for all questions without state lookup
- Access `input_data` fields via `.get()` with defaults (SDK schema stability concern from research open question 3)
