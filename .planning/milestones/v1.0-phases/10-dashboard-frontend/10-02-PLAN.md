  <files>

  <behavior>
    - Test: hook initializes with conductorState=null, events=[], connected=false
    - Test: on WebSocket open, connected becomes true
    - Test: on receiving a ConductorState snapshot message (has "version" field), conductorState is set
    - Test: on receiving a DeltaEvent message (has "type" field), event is appended to events array
    - Test: on DeltaEvent with type "agent_status_changed", the agent's status is updated in conductorState
    - Test: on DeltaEvent with type "task_status_changed", the task's status is updated in conductorState
    - Test: on DeltaEvent with type "agent_registered", new agent is added to conductorState.agents
    - Test: on DeltaEvent with type "task_assigned", task's assigned_agent is updated
    - Test: events array is capped at 500 entries (oldest evicted)
    - Test: on WebSocket close, connected becomes false
    - Test: sendIntervention sends JSON string over WebSocket
    - Test: on WebSocket close, reconnection is attempted with exponential backoff
  </behavior>
  <action>
    1. Create the reducer function handling four action types:
       - "snapshot": sets conductorState from full ConductorState JSON
       - "delta": appends event to events array (cap at 500, FIFO eviction), and calls applyDelta to update conductorState
       - "connected": sets connected=true
       - "disconnected": sets connected=false

    2. Implement applyDelta(state: ConductorState | null, event: DeltaEvent): ConductorState | null:
       - If state is null, return null (can't apply delta without snapshot)
       - agent_status_changed: find agent by event.agent_id, update status from payload.status
       - agent_registered: add new agent from payload to agents array
       - task_status_changed: find task by event.task_id, update status from payload.status
       - task_completed: find task, set status="completed"
       - task_failed: find task, set status="failed"
       - task_assigned: find task by event.task_id, set assigned_agent from payload.agent_id
       - Return new state object (immutable updates via spread)

    3. Create useConductorSocket(url: string) hook:
       - useReducer with reducer and initial state {conductorState: null, events: [], connected: false}
       - useRef for WebSocket instance, reconnect timeout, and retry count
       - connect() function: creates WebSocket, sets onopen/onmessage/onclose handlers
       - onmessage: use parseMessage from lib/messages.ts to determine snapshot vs delta
       - onclose: dispatch disconnected, schedule reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s)
       - useEffect: call connect() on mount, cleanup on unmount (clear timeout, close WebSocket)
       - sendIntervention(command: InterventionCommand): send JSON over WebSocket if connected
       - Return: { conductorState, events, connected, sendIntervention }

    4. For tests, mock WebSocket globally in test file. Create a MockWebSocket class that:
       - Tracks calls to send(), close()
       - Allows triggering onopen, onmessage, onclose from test code
       - Assign to globalThis.WebSocket before each test
       Use renderHook from @testing-library/react to test the hook.
  </action>
  <verify>
    <automated>cd packages/conductor-dashboard && npx vitest run src/hooks/useConductorSocket.test.ts</automated>
  </verify>
  <done>
    - Hook connects to WebSocket URL, dispatches connected/disconnected
    - Snapshot messages set full conductorState
    - Delta events are accumulated and applied to conductorState
    - Events array capped at 500
    - Exponential backoff reconnection on close
    - sendIntervention sends JSON over WebSocket
  </done>
