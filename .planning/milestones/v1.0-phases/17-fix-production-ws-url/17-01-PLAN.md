# Plan 17-01: Add Runtime Backend URL Configuration for Production WebSocket

## Goal
Make the npm dashboard package connect to the correct FastAPI backend in production by supporting runtime backend URL configuration, so the WebSocket connects to the FastAPI port instead of the sirv static file server port.

## must_haves
- Dashboard reads backend URL from `window.__CONDUCTOR_BACKEND_URL__` when present
- Dashboard falls back to same-origin URL when no backend URL is configured (dev mode)
- bin script accepts `--backend-url` CLI flag and injects it into served HTML
- TypeScript type declaration for the injected global
- npm package documentation covers production deployment

## Tasks

<task id="17-01-01" wave="1">
<title>Add runtime backend URL support to App.tsx and TypeScript declarations</title>
<description>
1. In `packages/conductor-dashboard/src/vite-env.d.ts`, add a TypeScript declaration for `window.__CONDUCTOR_BACKEND_URL__` as `string | undefined`
2. In `packages/conductor-dashboard/src/App.tsx`, replace the hardcoded same-origin WS_URL with a function that:
   - Checks `window.__CONDUCTOR_BACKEND_URL__` first
   - If set, derives WebSocket URL from it (replace http: with ws:, https: with wss:, append /ws)
   - If not set, falls back to current same-origin behavior: `${protocol}//${host}/ws`
3. Update `packages/conductor-dashboard/src/App.test.tsx` to test:
   - When `window.__CONDUCTOR_BACKEND_URL__` is set, the correct derived URL is used
   - When `window.__CONDUCTOR_BACKEND_URL__` is not set, same-origin fallback works
</description>

<task id="17-01-02" wave="2">
<title>Add --backend-url flag to bin script with HTML injection</title>
<description>
1. In `packages/conductor-dashboard/bin/conductor-dashboard.js`:
   - Parse `--backend-url <url>` from process.argv (alongside existing port arg)
   - If `--backend-url` is provided, create a custom HTTP request handler that:
     - For requests to `/` or `/index.html`, reads `dist/index.html`, injects a `<script>window.__CONDUCTOR_BACKEND_URL__ = "URL";</script>` tag before `</head>`, and serves the modified HTML
     - For all other requests, delegates to sirv as before
   - If `--backend-url` is not provided, use sirv directly (no injection, preserving current behavior)
   - Print the backend URL in the startup message when configured
2. Support both positional port arg and named flags: `conductor-dashboard [port] [--backend-url url]`
</description>
